## 1. 프로젝트 개요

### 1.1 배경

반도체 회사 **S-Semi**는 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 다양한 종류의 반도체 시료(Sample)를 생산·납품하고 있다.

시료는 고객 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

최근 주문량이 급증하면서 **엑셀과 메모장 기반의 수기 관리**로 인한 다음 문제들이 반복적으로 발생하고 있다.

| 문제 유형 | 구체적 사례 |
|-----------|------------|
| 주문 누락 | "어, 이 주문 처리됐나요?" |
| 공정 현황 불투명 | "공정 예약을 했는데, 언제 완성되는지 모르겠어요." |
| 불필요한 공정 가동 | "이미 충분한 시료 재고가 있는데, 왜 추가 공정이 돌아가고 있나요?" |

### 1.2 목적

위 문제를 해결하기 위해 **반도체 시료 생산 주문 관리 시스템**을 개발한다.

- 주문의 전체 생명주기를 시스템으로 추적·관리
- 재고 현황을 실시간으로 파악하여 불필요한 공정 가동 방지
- 생산 라인 스케줄을 투명하게 공개하여 납기 예측 가능
- 담당자 간 업무 경계를 명확히 하여 실수·혼선 최소화

## 2. 시스템 범위 및 제약 사항

| 항목 | 내용 |
|------|------|
| **운영 방식** | 콘솔(CLI) 기반으로 동작한다. GUI 없이 터미널에서 번호 입력으로 메뉴를 탐색한다. |
| **사용자** | 생산 담당자가 직접 명령을 입력하여 시료를 등록하고 주문을 처리한다. |
| **생산 라인 수** | 단일 생산 라인만 운영한다. 한 번에 하나의 시료만 생산 가능하다. |
| **생산 대상** | 생산 라인은 주문이 들어온 시료에 대해서만 생산을 수행한다. |
| **스케줄링** | 생산 대기 큐는 FIFO(선입선출) 방식으로 처리한다. |
| **데이터 저장** | 시스템 종료 시 데이터 영속성 보장 범위는 본 문서의 명세 범위 밖이다. |
| **외부 연동** | 고객과의 이메일 연동, 외부 시스템 API 연동은 범위 밖이다. |

## 3. 이해관계자 및 역할

### 3.1 역할별 흐름도

```
  [ 고객 ]
     |
     |  1. 시료 요청 (이메일)
     v
  [ 주문 담당자 ]
     |
     |  2. 주문서 작성 및 시스템 등록
     v
  [ 생산 담당자 ]
     |
     +-- 승인 --> 재고 확인 --> (재고 충분) CONFIRMED
     |                      --> (재고 부족) 생산 라인 등록 --> PRODUCING
     +-- 거절 --> REJECTED
```

### 3.2 역할 정의

| 역할 | 주요 책임 | 시스템 사용 기능 |
|------|-----------|----------------|
| **고객** | 필요한 시료를 이메일로 주문 담당자에게 요청한다. | 시스템 직접 사용 없음 |
| **주문 담당자** | 고객 요청을 받아 주문서를 작성하고 시스템에 등록한다. | 시료 주문 (시료 예약) |
| **생산 담당자** | 시료 등록, 주문 승인·거절, 생산 라인 관리, 출고 처리를 담당한다. | 시료 관리, 주문 승인/거절, 모니터링, 생산 라인 조회, 출고 처리 |

## 4. 주문 상태 흐름

### 4.1 상태 정의

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수 완료. 생산 담당자의 승인 대기 중 |
| `REJECTED` | 주문 거절. 정상 흐름에서 제외됨 (모니터링 미포함) |
| `PRODUCING` | 주문 승인 완료, 재고 부족으로 생산 라인에서 생산 중 |
| `CONFIRMED` | 재고 확보 완료, 출고 대기 중 |
| `RELEASED` | 출고 완료. 고객에게 전달됨 |

### 4.2 상태 전이 다이어그램

```
                     +-------------+
                     |   RESERVED  |  <-- 주문 담당자가 주문 등록
                     +------+------+
                            |
              +-------------+-------------+
           거절|                           |승인
              v                           v
       +------------+          재고 충분? 확인
       |  REJECTED  |         +------------+------------+
       +------------+      충분|                         |부족
         (흐름 종료)           v                         v
                       +-----------+           +-------------+
                       | CONFIRMED |           |  PRODUCING  |
                       +-----+-----+           +------+------+
                             |                        | 생산 완료
                             |                        v
                             |                 +-----------+
                             |                 | CONFIRMED |
                             |                 +-----+-----+
                             +----------+------------+
                                        | 출고 처리
                                        v
                                 +----------+
                                 | RELEASED |
                                 +----------+
```

### 4.3 상태 전이 조건 요약

| 전이 | 조건 |
|------|------|
| `RESERVED` -> `REJECTED` | 생산 담당자가 주문 거절 |
| `RESERVED` -> `CONFIRMED` | 생산 담당자가 승인 + 현재 재고 >= 주문 수량 |
| `RESERVED` -> `PRODUCING` | 생산 담당자가 승인 + 현재 재고 < 주문 수량 |
| `PRODUCING` -> `CONFIRMED` | 생산 라인에서 생산 완료 |
| `CONFIRMED` -> `RELEASED` | 출고 처리 실행 |

### 5.1 메인 메뉴

#### 개요

시스템 진입 시 가장 먼저 표시되는 화면으로, 전체 시료에 대한 요약 정보와 기능별 선택 메뉴를 제공한다.

#### 입력

- 사용자가 메뉴 번호를 입력한다.

#### 동작 조건

- 시스템 시작 시 자동으로 메인 메뉴를 표시한다.
- 각 기능을 번호로 선택하여 해당 기능 화면으로 이동한다.
- 유효하지 않은 번호 입력 시 오류 메시지를 출력하고 재입력을 요청한다.
- `0` 입력 시 시스템을 종료한다.

#### 출력

- 시료 현황 요약: 등록된 시료 수, 총 재고 수량, 상태별 주문 건수 (RESERVED, PRODUCING)
- 선택 가능한 메뉴 목록

#### 메뉴 구성

| 번호 | 메뉴명 |
|------|--------|
| 1 | 시료 관리 |
| 2 | 시료 주문 |
| 3 | 주문 승인 / 거절 |
| 4 | 모니터링 |
| 5 | 생산 라인 조회 |
| 6 | 출고 처리 |
| 0 | 종료 |

#### UI 예시

```
+======================================================+
|       S-Semi 반도체 시료 생산 주문 관리 시스템         |

|  총 재고 수량   : 128개                              |
|  처리 대기 주문 : 3건 (RESERVED)                     |
|  생산 중 주문   : 1건 (PRODUCING)                    |
+======================================================+
|  메뉴를 선택하세요.                                  |
|                                                      |
|  (1) 시료 관리                                       |
|  (2) 시료 주문                                       |
|  (3) 주문 승인 / 거절                                |
|  (4) 모니터링                                        |
|  (5) 생산 라인 조회                                  |

+======================================================+
|  선택 >> _                                           |
+======================================================+
```

### 5.2 시료 관리

#### 개요

시료(Sample)는 이 시스템의 가장 기본 단위다. 각 시료는 고유한 이름과 속성을 가지며, **시스템에 등록된 시료만 주문 가능**하다. 생산 담당자가 관리한다.

#### 공통 요구사항

| 번호 | 요구사항 |
|------|---------|
| SM-01 | 시료 ID는 시스템이 자동 부여하며 중복되지 않아야 한다. |
| SM-02 | 시료 이름은 필수 입력값이며 빈 값은 허용하지 않는다. |
| SM-03 | 평균 생산시간은 양수(0 초과)만 허용한다. |
| SM-04 | 수율은 0 초과 1 이하의 값(소수)이어야 한다. |
| SM-05 | 시료 조회 시 현재 재고 수량도 함께 표시한다. |
| SM-06 | 시료 검색은 이름을 기준으로 부분 일치를 지원한다. |

#### 시료 속성 정의

| 속성 | 설명 | 타입 | 예시 |
|------|------|------|------|
| 시료 ID | 시스템 자동 부여 고유 식별자 | 문자열 | `S001` |
| 이름 | 시료 명칭 | 문자열 | `GaAs-100` |
| 평균 생산시간 | 시료 1개 생산에 소요되는 평균 시간 (단위: 시간) | 실수 | `2.5` |
| 수율 | 공정 성공률 (0 < 수율 <= 1) | 실수 | `0.85` |

---

#### 5.2.1 시료 등록

**입력값:** 이름, 평균 생산시간, 수율

**동작 조건:**
- 이름이 빈 값이면 오류를 출력하고 재입력을 요청한다.
- 평균 생산시간이 0 이하이면 오류를 출력하고 재입력을 요청한다.
- 수율이 0 이하이거나 1 초과이면 오류를 출력하고 재입력을 요청한다.
- 유효한 입력이 완료되면 시료 ID를 자동 부여하고 시스템에 등록한다.

**출력:** 등록된 시료의 ID, 이름, 평균 생산시간, 수율 확인 메시지

**UI 예시**

```
[ 시료 관리 > 시료 등록 ]
---------------------------------------

  새로운 시료를 등록합니다.

  시료 이름        >> GaAs-100
  평균 생산시간(h) >> 2.5
  수율 (0~1)      >> 0.85

---------------------------------------
  [완료] 시료가 등록되었습니다.
    시료 ID : S003
    이름    : GaAs-100
    생산시간: 2.5h
    수율    : 85%

  엔터를 누르면 메인 메뉴로 돌아갑니다.
```

### 5.3 시료 주문

#### 개요

고객이 시료를 이메일로 요청하면 **주문 담당자**가 시스템에 주문을 등록한다. 주문이 등록되는 순간 상태는 `RESERVED`가 된다.

#### 요구사항

| 번호 | 요구사항 |
|------|---------|
| OR-01 | 주문 가능한 시료는 시스템에 등록된 시료에 한정된다. |
| OR-02 | 주문 수량은 1 이상의 정수여야 한다. |
| OR-03 | 고객명은 필수 입력값이며 빈 값은 허용하지 않는다. |
| OR-04 | 주문 등록 즉시 상태가 `RESERVED`로 설정된다. |
| OR-05 | 주문 ID는 시스템이 자동 부여한다. |

#### 입력값

| 속성 | 설명 | 조건 |
|------|------|------|
| 시료 ID | 주문할 시료의 ID | 시스템에 등록된 ID여야 함 |
| 고객명 | 주문을 요청한 고객의 이름 | 필수, 빈 값 불가 |
| 주문 수량 | 요청 수량 | 1 이상의 정수 |

#### 동작 조건

- 입력한 시료 ID가 시스템에 존재하지 않으면 오류를 출력하고 재입력을 요청한다.
- 모든 입력이 유효하면 주문 ID를 자동 부여하고 `RESERVED` 상태로 주문을 등록한다.

#### 출력

등록된 주문의 ID, 시료, 고객명, 주문 수량, 상태 확인 메시지

#### UI 예시

```
[ 시료 주문 > 시료 예약 ]
---------------------------------------

  주문할 시료 ID  >> S002
  고객명         >> 한국반도체연구소
  주문 수량      >> 10

---------------------------------------
  [완료] 주문이 접수되었습니다.
    주문 ID   : ORD-017
    시료      : SiC-300 (S002)
    고객명    : 한국반도체연구소
    주문 수량 : 10개
    상태      : RESERVED

  엔터를 누르면 메인 메뉴로 돌아갑니다.
```

**존재하지 않는 시료 ID 입력 시**

```
  주문할 시료 ID  >> S999
```

### 5.4 주문 승인 / 거절

#### 개요

**생산 담당자**가 `RESERVED` 상태의 주문 목록을 확인하고, 각 주문에 대해 승인 또는 거절을 결정한다. 승인 시에는 재고 상황에 따라 주문 상태가 자동으로 분기된다.

#### 요구사항

| 번호 | 요구사항 |
|------|---------|
| AP-01 | 목록에는 `RESERVED` 상태의 주문만 표시한다. |
| AP-02 | 승인 시 현재 재고를 자동으로 확인하여 상태를 분기한다. |
| AP-03 | 재고가 충분한 경우 (현재 재고 >= 주문 수량) 즉시 `CONFIRMED`로 전환한다. |
| AP-04 | 재고가 부족한 경우 (현재 재고 < 주문 수량) 생산 라인에 자동 등록하고 `PRODUCING`으로 전환한다. |
| AP-05 | 거절 시 즉시 `REJECTED`로 전환한다. |
| AP-06 | `RESERVED` 상태의 주문이 없을 경우 안내 메시지를 표시한다. |

#### 승인 시 자동 분기 로직

```
승인 선택
   |
   v
현재 재고 >= 주문 수량?
   +-- YES --> 상태: CONFIRMED
   +-- NO  --> 부족분 계산
                   |
                   v
               실 생산량 = ceil(부족분 / (수율 x 0.9))
               생산 라인 대기 큐에 등록
               상태: PRODUCING
```

#### 입력값

| 속성 | 설명 |
|------|------|
| 주문 ID | 처리할 주문의 ID |
| 처리 유형 | 승인(1) 또는 거절(2) 선택 |

#### UI 예시 - 주문 목록 확인


### 5.6 생산 라인 조회

#### 개요

현재 생산 중인 시료와 생산 대기 중인 큐를 표시한다. 생산 라인은 한 번에 하나의 시료만 처리하며, 대기 큐는 FIFO 방식으로 동작한다.

생산 완료 시 해당 주문의 상태가 `PRODUCING` -> `CONFIRMED`로 자동 전환된다.

#### 생산량 계산 공식

재고가 부족한 경우 아래 공식으로 실 생산량과 총 생산 시간을 계산한다.

```
실 생산량    = ceil( 부족분 / (수율 x 0.9) )
총 생산 시간 = 평균 생산시간 x 실 생산량
```

- 부족분: 주문 수량 - 현재 재고
- 수율에 0.9를 곱하는 이유: 공정 중 발생하는 불량률(10%)을 보수적으로 반영하여 충분한 수량을 생산하기 위함

#### 요구사항

| 번호 | 요구사항 |
|------|---------|
| PL-01 | 현재 생산 중인 시료 정보 (시료명, 주문 ID, 주문 수량, 실 생산량, 예상 완료 시간)를 표시한다. |
| PL-02 | 생산 대기 중인 큐 목록을 FIFO 대기 순서대로 표시한다. |
| PL-03 | 생산 완료 시 해당 주문의 상태를 `PRODUCING` -> `CONFIRMED`로 자동 전환한다. |
| PL-04 | 생산 라인과 대기 큐가 모두 비어있을 경우 안내 메시지를 표시한다. |

#### 입력값

없음 (조회 전용)

#### 출력

- 현재 생산 중인 시료 정보
- 생산 대기 큐 목록 (FIFO 순서)

#### UI 예시

```
[ 생산 라인 조회 ]
------------------------------------------------------------
  >> 현재 생산 중

  주문ID    시료       주문수량  실 생산량  예상 완료시간
  --------  --------  --------  --------  --------------
  ORD-017   SiC-300    10개      10개       55.0h 후

------------------------------------------------------------
  >> 생산 대기 큐 (FIFO)

  순번  주문ID    시료          주문수량  실 생산량
  ----  --------  ------------  --------  --------
   1    ORD-014   GaAs-100       8개       10개
   2    ORD-018   GaN-200        30개      50개

------------------------------------------------------------
  엔터를 누르면 메인 메뉴로 돌아갑니다.
```

**생산 라인이 비어있을 때**

```

---------------------------------------

  현재 생산 중인 항목이 없습니다.
```

### 5.7 출고 처리

#### 개요

`CONFIRMED` 상태의 주문에 대해 출고를 실행한다. 출고 처리 시 주문 상태는 `CONFIRMED` -> `RELEASED`로 전환된다.

#### 요구사항

| 번호 | 요구사항 |
|------|---------|
| RL-01 | 목록에는 `CONFIRMED` 상태의 주문만 표시한다. |
| RL-02 | 출고 처리 완료 시 상태를 `CONFIRMED` -> `RELEASED`로 전환한다. |
| RL-03 | `CONFIRMED` 상태의 주문이 없을 경우 안내 메시지를 표시한다. |

#### 입력값

| 속성 | 설명 |
|------|------|
| 주문 ID | 출고 처리할 주문의 ID |

#### 동작 조건

- 입력한 주문 ID가 `CONFIRMED` 상태가 아니면 오류를 출력한다.
- 유효한 주문 ID 입력 시 상태를 `RELEASED`로 전환하고 완료 메시지를 출력한다.

#### 출력

출고 처리된 주문의 ID, 시료, 고객명, 출고 수량, 상태 변경 확인 메시지

## 7. 데이터 모델

### 7.1 시료 (Sample)

시스템에 등록되는 반도체 시료를 나타내는 엔티티.

| 속성명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | 문자열 | Y | 시스템 자동 부여 고유 식별자 (예: `S001`) |
| name | 문자열 | Y | 시료 이름. 빈 값 불가 |
| avgProductionTime | 실수 | Y | 시료 1개 생산에 소요되는 평균 시간 (단위: 시간, 0 초과) |
| yield | 실수 | Y | 공정 성공률 (0 초과 1 이하) |
| stock | 정수 | Y | 현재 재고 수량 (0 이상). 초기값: 0 |

### 7.2 주문 (Order)

고객의 시료 주문을 나타내는 엔티티.

| 속성명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| id | 문자열 | Y | 시스템 자동 부여 고유 식별자 (예: `ORD-001`) |
| sampleId | 문자열 | Y | 주문한 시료의 ID (시료 엔티티 참조) |
| customerName | 문자열 | Y | 고객명. 빈 값 불가 |
| quantity | 정수 | Y | 주문 수량 (1 이상) |
| status | 열거형 | Y | 주문 상태: `RESERVED` / `REJECTED` / `PRODUCING` / `CONFIRMED` / `RELEASED` |
| createdAt | 일시 | Y | 주문 등록 일시 |

### 7.3 생산 작업 (ProductionJob)

생산 라인에서 처리되는 생산 작업을 나타내는 엔티티.

| 속성명 | 타입 | 필수 | 설명 |
|--------|------|------|------|
| orderId | 문자열 | Y | 연관된 주문의 ID |
| sampleId | 문자열 | Y | 생산할 시료의 ID |
| plannedQuantity | 정수 | Y | 실 생산량. `ceil(부족분 / (수율 x 0.9))`으로 계산 |
| totalProductionTime | 실수 | Y | 총 생산 시간. `평균 생산시간 x 실 생산량` |

### 7.4 엔티티 관계

```
Sample (1) ----< Order (N)
  - 하나의 시료는 여러 주문에 참조될 수 있다.

Order (1) ---- ProductionJob (0..1)
  - 재고 부족으로 승인된 주문은 생산 작업 1건을 가진다.
  - 재고 충분으로 승인된 주문은 생산 작업을 가지지 않는다.
```

## 8. 용어 정의

| 용어 | 설명 |
|------|------|
| **시료 (Sample)** | S-Semi에서 생산하는 반도체 시료. 주문·생산·출고의 기본 단위 |
