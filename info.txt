[프로젝트 배경]
반도체회사"S-Semi" 가 있습니다.
이회사는다양한종류의반도체시료(Sample)를 생산하여연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게
납품하고있습니다.
시료는주문이들어오면웨이퍼공정설비를통해제작되고, 검수를거쳐고객에게출고됩니다.
그런데최근들어주문량이급증하면서문제가생겼습니다.
"어, 이 주문 처리됐나요?"
"공정 예약을했는데, 언제완성되는지모르겠어요."
"이미 충분한시료재고가있는데, 왜추가공정이돌아가고있나요?“
엑셀과메모장으로주문을관리하다보니, 실수가잦고재고와공정현황을한눈에파악하기어려웠습니다.
이러한이유로S-Semi에서는 더체계적인시료관리를위한"반도체시료생산주문관리시스템"을개발하기로결정
했습니다

[역할별 흐름도]
역할은 고객, 주문담당자, 생산 담당자가 있어
고객은 주문담당자에게 시료요청을해. 주문담당자는 생산담당자에게 주문서를 전달해. 생산 담당자는 주문 담당자의 주문을 승인하거나 거절을해.
고객은 필요한 시료를 이메일로 요청해. 주문 담당자는 받은 요청에 맞게 주문서를 작성해. 생산 담당자는 개발 시료를 등록하고 주문을 수신하면 승인 또는 거절을 하는거지.

[시스템 개요]
생산 라인은 공장에서 시료 하나를 생산하는 설비의 흐름이야.
하나의 생산라인에서는 시료를 하나씩 생산해.
생산라인에서는 주문이 들어온 시료에 대해서만 생산해.
시스템은 콘솔 기반으로 동작해
생산 담당자가 직접 명령을 입력하고 시료 등록하고 주문을 처리해

[주문 상태 흐름]
모든 주문은 아래의 상태를 보유해.
상태와 의미를 순차적으로 말해보면
RESERVED - 주문접수, REJECTED - 주문 거절, PRODUCING - 주문 승인 완료 및 재고 부족으로 생산 중, CONFIRMED - 주문 승인 완료 및 출고 대기 중, RELEASE - 출고 완료
REJECTED는 거절된 주문으로 정상 흐름 외의 상태야. 이는 모니터링에서 제외해

즉, 주문 담당자로부터 주문이 등록되면 상태는 RESERVED가 돼. 
그 다음에 생산 담당자로부터 승인 여부에 따라 거절되면 REJECTED상태가 돼. 
승인이 된다면 그 다음으로는 재고 확인을 해. 
재고가 충분하다면 출고준비가 되고 상태는 CONFIRMED가 되고서 그 이후에 출고처리가 되어 상태는 RELEASED가 되는거지. 만약 재고가 부족했다면 
생산 라인으로 생산 요청이 들어간 뒤 상태는 PRODUCING이 되고 생산이 완료됐을때 순차적으로 CONFIRMED, RELEASED상태로 넘어갈거야. 그 다음에 고객에게 도달되는거지.


[기능 명세]
1. 메인 메뉴
*개요: 기능별 선택화면을 Display, 전체시료에대한 요약 정보를 확인
*메뉴:
(1) 시료 관리
- 새로운 시료 등록, 목록 조회, 이름 검색 기능
(2) 시료 주문
- 고객 주문 접수
(3) 주문 승인/ 거절
- 생산 라인 담당자의 승인·거절 처리
(4) 모니터링
- 상태별 주문수 및 시료별 재고 현황 확인
(5) 생산라인 조회
- 현재 생산 중인 시료 및 대기 중인 생산 큐 확인
(6) 출고 처리
- CONFIRMED 상태 주문에대해 출고 실행
(0) 종료

2. 시료 관리
*개요: 시료(Sample)는 이 시스템의 가장 기본이 되는 단위로 각 시료는 고유한 이름과 속성을 가지며, 시스템에 등록된 시료만 주문 가능함
*구성:
(1) 시료 등록
- 새로운 시료를 시스템에 추가. 속성 값: 시료 ID, 이름, 평균 생산시간, 수율
(2) 시료 조회
- 등록된 모든 시료 목록을 확인, 현재 재고 수량도 함께 표시
(3) 시료 검색
- 이름 등 속성으로 특정 시료를 검색

3. 시료 주문
*개요: 고객이 시료를 요청하면 주문 담당자가 주문을 생성.
*구성:
(1) 시료 예약
- 고객이 원하는 시료와 수량을 주문, 이 시점에서 주문 상태는 RESERVED
- 예약시 입력 값: 시료 ID, 고객명, 주문 수량

4. 주문 승인/거절
*개요: 접수된 주문(RESERVED) 목록을 확인, 특정 주문에 대하여 승인 혹은 거절
*구성:
(1) 접수된 주문 목록
- RESERVED 상태의 주문 목록 Display
(2) 주문 승인
- 접수된 특정 주문에 대해 승인. 이 때 승인시 재고 상황에 따라 2가지 방식으로 자동 처리 
- 재고가 충분한 경우에는 주문을 즉시 CONFIRMEDS 상태로 전환
- 재고가 부족한 경우에는 생산 라인에 자동으로 등록, 주문 상태를 PRODUCING으로 전환
(3) 주문 거절
- 접수된 특정 주문에 대해 거절, 즉시 REJECTED 상태로 전환

5. 모니터링 
*개요: 주문에 따른 시료 재고에 대한 정보 모니터링 기능, 담당자가 현재 시스템의 상태를 한 눈에 파악할 수 있도록 구성
* 구성:
(1) 주문량 확인
- 현재 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASED) 목록을 확인
- REJECTED는 유효한 주문이 아니므로 무시
(2) 재고량 확인
- 각 시료별 현재 재고 수량을 확인
- 주문대비 재고 수량에 따라 상태도 표기. 
- 여유: 주문대비 재고 충분 상태, 부족: 주문대비 재고 수량 부족 상태, 고갈: 수량이 0인 상태

6. 생산 라인 조회
*개요: 생산 라인에 대한 정보를 Display. 주문량에 대한 부족분을 생산하되, 수율 오차를 고려하여 시료를 생산. 
- 실 생산량: ceil(부족분/(수율 *0.9))
- 총 생산 시간: 평균 생산시간 * 실 생산량 
- 생산 완료시 주문상태 PRODUCING -> CONFIRMED 변경
*구성:
(1) 생산 현황 표기
- 현재 생산 중인 시료에 대한 정보 표기
(2) 대기 주문 확인
- 생산 라인의 대기열인 생산 큐를 이용, 생산 작업을 대기하고 있는 목록을 출력. (스케줄링 전략: FIFO)

7. 출고 처리
*개요: 재고가 충분해진 CONFIRMED 주문에 대하여 출고를 처리.  특정 주문에 대해 출고를 실행, 주문 상태가 RELEASE로 전환
